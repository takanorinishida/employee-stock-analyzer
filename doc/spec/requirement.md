# 持株会管理ツール 要求仕様書

バージョン: 1.1.0
作成日: 2026-06-24
ステータス: ドラフト

---

## 1. 機能要求

| 要求ID | 要求カテゴリ | 要求内容 | 要求理由 |
|--------|------------|---------|---------|
| REQ-0001 | 機能要求 | ユーザーは持株会を複数登録・管理できること | 転職等により複数社の持株会に加入する場合があるため |
| REQ-0002 | 機能要求 | ユーザーは持株会の情報（会社名・証券コード等）を編集できること | 会社情報の変更や誤入力を修正できるようにするため |
| REQ-0003 | 機能要求 | ユーザーは持株会の拠出情報（拠出金・奨励金・取得株数）を記録できること | 毎月の積立購入を正確にトラッキングするため。奨励金は会社補助として取得コストに反映させるため |
| REQ-0004 | 機能要求 | ~~廃止~~ REQ-0003に統合 | — |
| REQ-0005 | 機能要求 | ユーザーは配当金による再投資を拠出として記録できること | 持株会では配当金が自動的に株式購入に再投資される独立したイベントであるため |
| REQ-0006 | 機能要求 | ユーザーは株式の売却情報（売却日・株数・売却単価）を記録できること | 売却によって確定した損益を管理するため |
| REQ-0007 | 機能要求 | ユーザーは株式分割・合併イベントを記録できること | コーポレートアクションにより保有数・単価が変動するため |
| REQ-0008 | 機能要求 | システムは奨励金・配当金を取得コストに含めた移動平均法により平均取得単価を自動計算すること | 取引のたびに正確な取得コストを把握するため |
| REQ-0009 | 機能要求 | システムは奨励金・配当金を取得コストに含めない場合の平均取得単価を計算できること | 会社補助・配当金を利益として捉えた場合の取得コストを把握するため |
| REQ-0010 | 機能要求 | システムは実際に売却した際の売却損益（（売却単価 − 平均取得単価）× 売却株数）を自動計算すること | 確定した損益をリアルタイムで把握するため |
| REQ-0011 | 機能要求 | システムは実際に売却した際の売却益に対する譲渡所得税の概算（20.315%）を自動計算すること | 売却後の確定税額を把握するため |
| REQ-0012 | 機能要求 | ユーザーは現在の保有株数・平均取得単価・累積拠出額をサマリーとして確認できること | 資産状況を俯瞰的に把握するため |
| REQ-0013 | 機能要求 | ユーザーは取引履歴を分析・共有用途のCSVファイルとして出力できること | 外部ツール（Excelなど）での分析や他者との共有に対応するため |
| REQ-0014 | 機能要求 | ユーザーは規定フォーマットのCSVファイルから取引履歴をインポートできること | 過去データの一括登録や他ツールからの移行を容易にするため |
| REQ-0015 | 機能要求 | ユーザーは全データを復元目的の完全バックアップとしてエクスポート・インポートできること | データ損失時に完全な状態へ復元できるようにするため |
| REQ-0016 | 機能要求 | ユーザーは売却前のシミュレーションとして現在の株価を入力することで、評価損益・概算税額・手取り額を確認できること | 売却判断の前に手取り額と評価損益を把握できるようにするため |

## 2. 非機能要求

| 要求ID | 要求カテゴリ | 要求内容 | 要求理由 |
|--------|------------|---------|---------|
| REQ-1001 | 非機能要求 | システムは平均取得単価の計算において浮動小数点誤差が生じない精度を保証すること | 金融計算において誤差が損益の誤算につながるため |
| REQ-1002 | 非機能要求 | システムは保有株数を小数点以下4桁まで管理できること | 持株会では端株（1株未満の株式）が発生するため |
| REQ-1003 | 非機能要求 | 譲渡所得税率（20.315%）はシステム内で定数として一元管理されること | 将来の税制変更に対して修正箇所を最小化するため |
| REQ-1004 | 非機能要求 | 取引履歴を修正・削除した場合、それ以降の全計算結果が自動的に再計算されること | 過去の修正が後続の計算に矛盾なく反映されるようにするため |
| REQ-1005 | 非機能要求 | CSVインポート時はバリデーションを実施し、エラー内容を全件ユーザーに通知すること | 誤ったデータの取り込みを防ぐため |
| REQ-1006 | 非機能要求 | UIは日本語で表示されること | 主な利用者が日本語話者であるため |
